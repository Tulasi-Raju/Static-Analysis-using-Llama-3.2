Hospital Management System

The hospital management system is a Java-based application that allows users to manage doctors, patients, and examination rooms. It provides an interface for adding, checking in, checking out, and managing patients and doctors.

Doctor.java

The Doctor class represents a doctor in the hospital. It has fields for the doctor's first name, last name, birth date, specialty, favorite room, and checked-in status. The class includes methods for setting these fields and checking if the doctor is already checked in. * What this file/class is for: The Doctor class is used to represent a doctor in the hospital. * Important fields and methods: The doctor's first name, last name, birth date, specialty, favorite room, and checked-in status are important fields. Methods like getSpecialty() and setCheckedIn(boolean) allow you to access and modify these fields. * How it interacts with other files/classes: The Doctor class interacts with the ExaminationRoom class by storing a doctor in an examination room.

ExaminationRoom.java

The ExaminationRoom class represents an examination room in the hospital. It has fields for the room's number, occupied status, running status, occupying doctor, occupying patient, and waiting patients. The class includes methods for comparing sizes of waiting lists and resetting the examination room. * What this file/class is for: The ExaminationRoom class is used to represent an examination room in the hospital. * Important fields and methods: The room's number, occupied status, running status, occupying doctor, occupying patient, and waiting patients are important fields. Methods like compareTo(ExaminationRoom o) compare sizes of waiting lists and reset the examination room. * How it interacts with other files/classes: The ExaminationRoom class interacts with the Doctor class by storing a doctor in an examination room.

Hospital.java

The Hospital class represents the hospital itself. It has fields for the list of doctors, patients, and examination rooms. The class includes methods for initializing the hospital, adding examination rooms, checking in patients and doctors, checking out patients and doctors, updating waiting lists, getting running rooms, and getting examination rooms. * What this file/class is for: The Hospital class is used to represent the hospital itself. * Important fields and methods: The list of doctors, patients, and examination rooms are important fields. Methods like initializeHospital() add examination rooms, checkInPatient(Patient patient) checks in a patient, checkOutDoctor(Doctor doctor) checks out a doctor, updateWaitingLists(), getRunningRooms(), and getExaminationRooms() allow you to manage the hospital. * How it interacts with other files/classes: The Hospital class interacts with the ExaminationRoom class by storing examination rooms and updating waiting lists.

Example Use Cases

* Add a new doctor to the hospital: 1. Create an instance of the Doctor class with the doctor's details. 2. Call the checkInDoctor() method on the Hospital class with the doctor instance as an argument. * Check in a patient to the hospital: 1. Create an instance of the Patient class with the patient's details. 2. Call the checkInPatient() method on the Hospital class with the patient instance as an argument. * Check out a doctor from the hospital: 1. Get the list of doctors in the hospital using the getDoctors() method on the Hospital class. 2. Select the desired doctor and call the checkOutDoctor() method on the Hospital class with the selected doctor as an argument. * View patient information: 1. Get the list of patients in the hospital using the getPatients() method on the Hospital class. 2. Select the desired patient and view their details.
